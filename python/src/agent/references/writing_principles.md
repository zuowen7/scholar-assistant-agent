# 学术写作原则

使用本文件理解不同论文章节的修辞任务，不要仅按原文顺序逐句翻译。

## 论文类型识别

在编辑/翻译前，识别论文类型：

- `Research paper`: 读者问"为什么这个现象重要？做了什么？发现了什么？意味着什么？"
- `Methods paper`: 读者问"方法是否有效？是否可复现？是否优于现有方法？"
- `Review`: 读者问"这个领域现状如何？有哪些争议？未来方向是什么？"

## 章节职责

### Introduction
- 告诉读者为什么这项工作重要
- 解释填补了什么知识缺口
- 陈述已知和未知
- 说明论文提出的问题和如何解决
- **不要**在此总结 Results 或 Conclusion

### Results
- 报告观察到的内容（过去时为主）
- 说明在什么条件下、有什么定量支持
- **不要**在此做长篇机制解释

### Discussion
- 解释发现意味着什么
- 与早期工作比较（支持/复杂化/修正）
- 陈述局限性
- 使用适当的 hedging

### Methods
- 确保可复现性
- 拒绝模糊措辞："标准条件"、"常规方法"、"数据经统计分析"

### Conclusion
- 重述核心贡献
- 总结决定性证据
- 陈述含边界的含义
- 不引入新数据

## 过度宣称检测

自动标记并要求修正以下词汇（除非有强证据支持且范围明确）：

| 过度宣称 | 建议替换 |
|---------|---------|
| prove | show / demonstrate |
| conclusively | — |
| unprecedented | to our knowledge |
| best | among the strongest |
| first (unqualified) | initial / among the first |

## AI 使用边界

- 允许：改善语法/清晰度、提供结构建议、术语和 hedging 检查
- 谨慎：解释方法/结果措辞、起草审稿回复框架
- 不允许：AI 起草核心科学论证、编造参考文献
