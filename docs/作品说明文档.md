# 研墨 Scholar Assistant — 参赛作品说明文档

> **作品名称**：研墨 Scholar Assistant
> **作品定位**：全球首款"翻译—思考—写作—投稿"全链路一体化的隐私优先学术 AI 工作站
> **版本**：Tauri 0.3.1 / npm 0.2.1
> **形态**：Tauri 2 桌面应用（Win / macOS / Linux）+ Docker 镜像 + Python CLI
> **目标用户**：研究生、青年科研工作者、跨国合作研究团队、高校实验室

---

## 一、产品目标 — 我们到底要解决什么问题

### 1.1 现状与痛点

科研工作者从"读外文文献"到"投稿英文期刊/会议"，**真实使用的工具链长达 7–10 个**：

| 环节 | 当前主流工具 | 痛点 |
|------|------------|------|
| PDF 翻译 | DeepL / 谷歌翻译 / 沉浸式翻译 | 通用翻译，**不识别公式/双栏/引用区**；术语前后不一致；隐私顾虑 |
| 文献管理 | Zotero / Mendeley | 与翻译、写作工具完全割裂 |
| AI 润色 | ChatGPT / Notion AI / Grammarly | **数据上云**，未发表成果泄露风险；非学术语境，提示词需反复调教 |
| 思路梳理 | XMind / 幕布 | 思维导图与论文结构无法转换，需手动重写 |
| 论证组织 | **市面无对应工具** | 论文逻辑链全靠脑力维持，子论点容易遗忘 |
| 论文排版 | Overleaf + LaTeX | 模板查找耗时，与写作工具脱节 |
| 引用管理 | EndNote / BibTeX | 与全文写作场景割裂 |

**核心痛点三连：**
1. **工具割裂**：研究者一天要切 7 个软件，思路在切换中持续流失。
2. **数据泄露风险**：未发表成果上传到 ChatGPT、DeepL Pro 是当前科研伦理的灰色地带，国内多所高校已明令禁止。
3. **学术不原生**：通用 AI 工具不懂学术——不会跳过引用区、不会保持术语一致、不会按 IEEE/ACM 模板排版、更不会组织论证逻辑。

### 1.2 我们的目标

**用一款隐私优先的桌面应用，把"读 → 想 → 写 → 投"四个环节缝合成一条流水线**，让研究者在同一个窗口内完成从 PDF 文献到投稿就绪的 LaTeX/Word 全流程，**全程数据不离开本地机器**。

## 二、产品创新 — 我有别人无的核心壁垒

### 🏆 创新点 ①：动态论证地图（Dynamic Argument Mapping）— **业界首创**

> **这是研墨的"杀手锏"，市面上没有任何一款 AI 写作工具具备同等能力。**

**问题**：写论文最难的不是"语言"，是"论证链组织"——哪些论点支撑哪些论点？哪些证据绑定哪个论点？哪里有逻辑跳跃？现有 AI 写作工具（Notion AI、Jasper、Cursor）全部停留在**"段落级润色"**层面，对论证结构无能为力。

**研墨方案**（`python/src/argument/`）：将论文的**论证结构本身**作为一等公民数据结构：

| 模块 | 文件 | 能力 |
|------|------|------|
| 论证树 CRUD | `store.py` + `models.py` | 节点 = 论点；边 = 支撑/反驳关系；叶子可绑 RAG 文献作证据 |
| AI 智能扩展 | `expander.py` | 选中父节点，LLM 自动生成 3–5 个子论点 |
| 逻辑审查 | `logic_checker.py` | AI 全树扫描循环论证、跳跃推理、证据缺失 |
| 观测反馈 | `observer.py` + `feedback_generator.py` | 每个节点可挂"评审意见"和"反例" |
| **降维展开** ⭐ | `flatten.py` | **DFS 遍历论证树，LLM 逐节点扩写为 150–300 字学术段落，注入挂载文献的 `[citation_key]`，SSE 流式输出 Markdown / LaTeX / DOCX 论文初稿** |

**降维展开**是真正意义上的"思路 → 论文"管道——这是市面上唯一把**抽象论证结构**直接变成**结构化论文**的引擎。

---

### 🏆 创新点 ②：三视图同构 — Mindmap ↔ ArgumentMap ↔ Markdown 双向转换

研究者在不同思考阶段需要不同的视图：发散期要思维导图，收敛期要论证树，撰写期要 Markdown。**研墨让同一份知识在三种视图间无缝切换**：

- **思维导图视图**（Vue Flow + dagre 自动布局）：发散思考、自由连线
- **论证树视图**：聚焦逻辑、绑定证据
- **Markdown 视图**：撰写润色

**业界对比**：XMind 只能导出图片；幕布只能转大纲；Notion 没有图形视图——**没有任何工具实现"三视图同构"**。

---

### 🏆 创新点 ③：五阶段 SSE 翻译流水线 + 块级失败重试 — 工程级而非"调用 LLM"

**业界主流翻译方案对比**：

| 方案 | DeepL Pro | 沉浸式翻译 | 知云翻译 | ChatGPT | **研墨** |
|------|-----------|-----------|---------|---------|---------|
| PDF 双栏自动检测 | ✅ | ❌ | 部分 | ❌ | ✅ |
| 引用区跳过不翻译 | ❌ | ❌ | ❌ | ❌ | ✅ |
| 17 阶段文本清洗（连字符断词/cid 残留/水印过滤） | 部分 | ❌ | ❌ | ❌ | ✅ |
| **失败块单独重译**（无需重新翻译全文） | ❌ | ❌ | ❌ | ❌ | ✅ **独有** |
| **滑动术语表自动注入**（保证术语一致性） | ❌ | ❌ | ❌ | ❌ | ✅ **独有** |
| 句子级悬停高亮对照 | ✅ | ✅ | ❌ | ❌ | ✅ |
| 全本地离线运行 | ❌ | ❌ | ❌ | ❌ | ✅ |
| 5 步 SSE 实时进度推送 | ❌ | ❌ | ❌ | ❌ | ✅ |

**核心创新**：将翻译拆解为 `Parse → Clean → Chunk → Translate → Format` 五个**可独立诊断、可单独重试**的阶段。一篇 50 页论文翻译到第 80% 失败，传统方案要重头开始，研墨只需点击重译失败的块。

---

### 🏆 创新点 ④：21 家云端 LLM 统一抽象层 — 业界覆盖最广

**对比业界其他翻译/写作工具**：

| 工具 | 支持的 LLM 数量 |
|------|----------------|
| ChatGPT | 1（仅 OpenAI） |
| Notion AI | 1（仅自家） |
| Cursor | ≈ 5（GPT/Claude/Gemini） |
| 沉浸式翻译 | ≈ 8 |
| **研墨** | **21 家 + 本地 Ollama** |

研墨在 `cloud_client.py::PROVIDER_PRESETS` 中预置 **21 家**云端供应商参数：
OpenAI / Anthropic / DeepSeek / Moonshot / 智谱 / 通义千问 / Gemini / SiliconFlow / OpenRouter / Groq / Together / Mistral / xAI / Fireworks / DeepInfra / Perplexity / Novita / 火山方舟 / 百度千帆 / Azure OpenAI / 自定义 OpenAI 兼容端点。

**用户价值**：当 OpenAI 限流、API 涨价、被封号时，**一键切换**到 DeepSeek / 通义千问 / 国产大模型，不需要换软件。这是真正的"模型自由"。

### 🏆 创新点 ⑤：隐私优先的真·全离线运行

**这是当前国内高校与科研院所最稀缺的能力**——多所"双一流"高校已明令禁止将未发表成果上传到 ChatGPT、Claude、DeepL Pro 等境外服务。

**研墨的隐私架构**：

```
                ┌────────── 用户机器 ──────────┐
                │                              │
   PDF 文献 ─→  │  Tauri 桌面应用（壳）         │
                │     ↓                        │
                │  Python 后端（localhost:18088）│
                │     ↓                        │
                │  Ollama 本地推理（11434）      │
                │     ↓                        │
                │  ChromaDB 本地向量库          │
                │                              │
                └─────────────────────────────┘
                  全程零联网，零数据外发
```

- 桌面端关闭时通过 `taskkill /F /T` 杀掉进程树，零残留
- CSP 严格限制 `connect-src` 仅允许 `localhost`，**Webview 物理上无法发起外部请求**
- 本地路径访问沙箱化（`_validate_file_path()`），拒绝访问 `~/.ssh`、`~/.aws`、`AppData\Roaming` 等敏感目录
- API Key 自动 Mask，写盘前自动剥离

**业界对比**：DeepL Pro / Notion AI / ChatGPT 全部强制上云；Cursor 即使所谓"Privacy Mode"也要联网；唯一全离线的方案就是研墨 + Ollama。

---

### 🏆 创新点 ⑥：Agent 自演化系统 — Skill 从过往任务自动沉淀

**业界对比**：ChatGPT / Claude 的"记忆"是简单的对话历史；Cursor 的 `.cursorrules` 需要手写；只有 Devin 等少数顶级 Agent 工具具备从轨迹中**自动提取技能**的能力。

**研墨方案**（`python/src/agent/`）：

```
任务执行 → Trajectory 记录 → ReviewAgent 评审 → SkillSystem 提取
                                                      ↓
                                               技能库 (JSON 持久化)
                                                      ↓
                          下一次任务 → 多信号匹配 (关键词/嵌入/类别) → 自动注入
```

- `_skill_auto.py`：从 Trajectory 自动识别可复用模式
- `_skill_matching.py`：多信号匹配下发
- `_skill_model.py` / `_skill_persistence.py`：模型与持久化
- `review_agent.py`：事后评审打分

**用户体感**：用得越久越聪明——第一次让它"格式化引用"需要详细描述，第三次只要说"按上次的方式做"。

## 三、市场价值与社会意义

### 3.1 目标用户规模

- 中国研究生在校规模 **>365 万**（2024 教育部数据）
- 高校教师 **≈ 200 万**
- 科研院所研究人员 **>300 万**
- **核心目标用户基数：约 800 万人**
- 全球非英语母语研究者 **>2000 万**（潜在国际化市场）

### 3.2 商业模式潜力

| 模式 | 说明 |
|------|------|
| 个人版（社区） | 桌面端免费，使用本地 Ollama 全部功能 |
| 个人版（Pro） | 解锁云端 21 家供应商接入、高级模板、PPT 导出 |
| 高校实验室版 | Docker 私有化部署，全实验室共享 RAG 知识库 |
| 出版社/期刊版 | 集成投稿前自动合规检查（字数、图表、引用格式） |

### 3.3 社会价值

1. **降低非英语母语研究者投稿门槛**：让更多优秀的中国研究成果走向国际期刊。
2. **保护科研数据隐私**：响应国家关于科研数据安全的号召，符合《数据安全法》《个人信息保护法》。
3. **提高科研效率**：把研究者从"工具切换"中解放出来，每天可节省 1–2 小时。
4. **促进国产大模型生态**：默认接入 DeepSeek、智谱、通义、火山等国产 LLM。

## 四、技术壁垒（评委视角）

### 4.1 工程复杂度

- **代码规模**：Python 后端 ~30,000 行 + Vue 前端 ~25,000 行 + Rust 桌面壳 ~500 行
- **测试覆盖**：后端 38+ 单元测试 / 9 个集成测试，前端 17 个 Vitest 测试
- **架构层级**：5 个 FastAPI Router 模块 + 16 种 PDF 解析器 + 17 阶段清洗管线 + 21 家 LLM 客户端 + 5 类工具集

### 4.2 难度集中点

1. **PDF 解析的"脏数据"清洗**：连字符断词、`cid:N` 残留、pdfplumber 字体编码损坏、双栏识别——每一项都是踩了无数次坑后沉淀的工程经验。
2. **块级翻译的术语一致性**：滑动术语表 + 正则提取 `中文(English)` 模式 + 上下文窗口注入——这套机制是研墨长文档术语翻译质量的核心。
3. **Agent 的可恢复性**：Session 持久化 / 断点续传 / 人工审批 / 撤销上一步——把生产级 Agent 的所有可靠性手段全部实现。
4. **论证树降维展开**：DFS 遍历 + LLM 逐节点扩写 + 引用注入 + SSE 流式——这是从抽象结构到具体文本的**端到端管道**。
5. **桌面端进程编排**：Tauri 同时管理 Python 后端 + Ollama + 健康检查 + Windows 代理变量清理（httpx import 卡顿问题）。

### 4.3 创新性总结（一句话）

> **研墨的核心创新不是单个功能，而是把"五阶段翻译管线 + 论证地图引擎 + ReAct Agent + Skill 自演化 + 21 家 LLM 抽象层 + 全离线架构"组合成业界唯一的学术 AI 工作站。**
