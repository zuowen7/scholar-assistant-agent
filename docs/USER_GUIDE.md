## 安装与启动

### 前提条件

安装 [Ollama](https://ollama.com) 并拉取模型（本地翻译可选）：

```bash
ollama pull qwen3:8b
```

> 不装 Ollama 也行——默认使用云端引擎，只需在设置里填 API Key。

### 方式一：桌面端（推荐）

1. 从 [GitHub Releases](https://github.com/zuowen7/scholar-assistant/releases) 下载安装包
2. 安装后启动即可，应用自动管理后端服务
3. 应用启动时会自动检查新版本，有更新会弹 Toast 提示

**Windows 开发者**：使用 `start_dev.bat` 启动（自动清除代理环境变量，避免卡顿）

### 方式二：开发模式

```bash
# 终端 1 — 启动 Python 后端
cd python
pip install -r requirements.txt
python api.py

# 终端 2 — 启动前端
npm install
npm run dev
```

浏览器打开 `http://localhost:5173`，前端自动代理 `/api` 到 `localhost:18088`。

### 方式三：Docker

```bash
docker compose up
```

## 翻译模式

顶栏切到「翻译」即可进入。拖入 PDF，等待完成。

### 翻译一篇论文

1. **上传文件**：点击上传区域选择 PDF，或直接把文件拖入窗口任意位置
2. **等待处理**：5 步管道自动执行，进度条实时显示
   - 解析 → 清洗 → 切块 → 翻译 → 格式化
3. **查看结果**：翻译完成后进入结果页面

### 结果视图

| 视图 | 说明 |
|------|------|
| **双语对照** | 左原文右译文，鼠标悬停某个句子，另一侧对应句子高亮 |
| **纯译文** | 只显示翻译后的文本 |
| **Markdown** | 渲染为格式化的 Markdown 页面 |

### 翻译失败重试

如果某段翻译失败（红色标记），不用重新翻译全文：
- 点击失败的段落 → 选择「重试」→ 仅重新翻译该段

### 导出翻译结果

右上角下拉菜单，4 种格式：

| 格式 | 内容 |
|------|------|
| 双语 Markdown | 中英交替，段落对齐 |
| 纯译文 Markdown | 只保留中文翻译 |
| 双语 Word (.docx) | 中英对照，适合打印 |
| 纯译文 Word (.docx) | 纯中文版本 |

### 翻译后自动入库

翻译完成后，双语全文自动存入本地文献库（ChromaDB 向量数据库）。之后在 Agent 对话中可以用「搜索文献」工具检索之前翻译过的内容。

## 编辑器模式

顶栏切到「编辑器」进入。核心是 Monaco 编辑器 + 左侧文件树 + 右侧面板。

### 创建 / 打开项目

进入编辑器后，欢迎页提供 4 个入口：

| 入口 | 用途 |
|------|------|
| **新建工程** | 从思维导图开始，一步步生成论文草稿 |
| **从模板新建** | 选 IEEE / ACM / NeurIPS 等模板，AI 生成大纲 |
| **打开文件夹** | 打开本地已有的论文项目（Agent 工作区） |
| **新建文档** | 创建空白 Markdown 文件 |

### 文件管理

- 左侧**文件树**：浏览、新建、删除文件和文件夹
- 顶部**标签栏**：多文件切换，拖拽排序，`Ctrl+W` 关闭当前标签

### Ghost Text（自动续写）

在编辑器里打字后**停顿 1.5 秒**，AI 会在光标处生成灰色续写建议：
- **Tab** — 接受建议
- **Esc** — 忽略

### 预览

右侧面板切到「预览」，实时渲染当前 Markdown 为排版效果，所见即所得。

### 语音输入

编辑器工具栏提供麦克风按钮，支持语音打字：

- 点击 🎤 → 开始说话，实时转录到编辑器光标位置
- 再次点击 → 停止录音
- 说话过程中按 `Tab` 接受 Ghost Text 补全后，后续语音自动拼接

## 语音助手

语音助手让你像用 Siri 一样控制研墨——喊「小研」或按热键，说指令，自动执行。

### 激活方式

| 方式 | 操作 |
|------|------|
| **唤醒词** | 说「**小研**」（默认），弹出语音界面 |
| **全局热键** | 按 `Alt+Shift+V`，即使窗口最小化也能唤醒 |

> 唤醒词仅在应用窗口在前台时有效；全局热键系统级生效（仅桌面端）。

### 语音界面

激活后弹出 Siri 风格全屏界面：

- **脉冲球体** — 呼吸动画，提示正在聆听
- **波纹扩散** — 3 层同心圆波纹，表示麦克风活跃
- **实时转写** — 你说的文字逐字出现在中央
- **静默自动提交** — 停顿 2 秒后自动发送给 Agent 执行
- **10 秒无语音** — 自动取消

> 按 `Escape` 或点击遮罩区域取消语音指令。

### 使用流程

1. 喊「**小研**」或按 `Alt+Shift+V` → 语音界面弹出
2. 说出你的指令，例如：
   - "帮我翻译当前段落"
   - "把实验部分写得更详细"
   - "搜索论文中关于 transformer 的内容"
3. 停顿 2 秒 → 自动提交给 Agent 执行
4. Agent 在侧栏面板中执行，语音界面自动关闭

### 唤醒词支持

默认唤醒词为「**小研**」。语音识别支持同音字变体：

- 小研 / 小严 / 小言 / 小岩 / 小颜 等均能识别
- 5 秒冷却机制：唤醒后 5 秒内再次说唤醒词不会重复触发

### 语音打字

编辑器工具栏、Agent 面板、AI 编辑面板均有麦克风按钮：

- 点击 → 开始语音输入，说话即转录为文字
- 再点击 → 停止录音
- 与唤醒词互不干扰：语音打字时唤醒词自动暂停
- 多次说话之间由 Chrome 自动去重，不会出现重复文本

### 语音设置

顶栏设置面板 →「语音」标签页：

| 设置项 | 说明 | 默认值 |
|--------|------|--------|
| 启用语音助手 | 开启/关闭唤醒词+热键 | 开 |
| 唤醒词 | 自定义唤醒词（2-4 个字） | 小研 |
| 全局热键 | 按键捕获，自定义热键 | Alt+Shift+V |
| 语音语言 | 识别语言 | zh-CN |
| 灵敏度 | 唤醒词检测灵敏度 | 中 |

## AI Agent 对话

右侧面板切到「Agent」，这是项目的核心——像 Claude Code 一样直接操作你的工作区文件。

### 基本用法

1. 先通过「打开文件夹」指定一个项目目录
2. 在 Agent 对话框输入指令，例如：
   - "帮我读一下 introduction.md，总结要点"
   - "把第 3 段的实验描述改写得更严谨"
   - "搜索文献库中关于 transformer 的论文"
3. Agent 会读取、修改你的文件，每次修改后编辑器自动刷新

### 快速操作（底部预设按钮）

| 按钮 | 效果 |
|------|------|
| 润色 | 改善选中段落的行文质量 |
| 扩写 | 将选中段落展开为更详细的论述 |
| 审查 | 检查逻辑漏洞、论证不足之处 |
| 英译 | 将选中中文翻译为学术英文 |
| 中译 | 将选中英文翻译为中文 |

> 这些预设按钮走一次性 LLM 流式输出，不经过 Agent ReAct 循环，响应更快。

**使用方式**：
1. 在编辑器里**选中一段文字**
2. 点击预设按钮（或在输入框直接打字提要求）
3. AI 返回结果，点击「**插入**」替换选中文字，或「**复制**」到剪贴板
4. 不满意？点击「**撤销**」恢复原文

### 高级功能

- 输入 `/` 呼出斜杠命令菜单
- 输入 `@` 引用工作区其他文件作为上下文
- 点击 📎 按钮添加附件
- Agent 读 PDF 会自动提取纯文本，不需要手动转换

### 越界审批

Agent 的文件操作被限制在项目目录内。如果它需要访问项目外的文件（比如系统配置），会弹窗请求你批准：

| 选项 | 含义 |
|------|------|
| 允许本次 | 只批准这一次操作 |
| 本次会话允许 | 整个会话期间不再询问 |
| 拒绝 | 拒绝操作 |

### 文献库（RAG）

Agent 面板中有「文献库」标签页：
- 翻译过的论文会自动入库
- 可以手动上传文件或删除已有文献
- Agent 通过 `search_documents` 工具按需检索，不会每轮自动注入

## 论证陪练

右侧面板切到「论证陪练」，这是投稿前的自查工具，分两个子页签。

### 论证账本

**用途**：AI 自动检查你在 abstract/intro 里许下的每一条承诺，是否在正文中实际兑付了。

**操作步骤**：

1. 打开你的论文文件
2. 点击「**分析论证账本**」按钮
3. AI 扫描后，每条承诺显示为一行：

| 状态 | 含义 |
|------|------|
| ✅ 已兑付 | 正文有对应实验/论证支撑 |
| ⚠️ 部分兑付 | 有支撑但不充分 |
| ❌ 未兑付 | 正文找不到对应的实验/论证 |
| ⚠️ 不一致 | 兑现了但和承诺的内容对不上 |

4. **点击承诺文本** → 编辑器跳转到对应原文位置
5. 点击「**→ 兑付处**」→ 跳到正文中支撑该承诺的位置
6. 对未兑付的承诺点「**怎么补满**」→ AI 给出具体实验设计建议
7. **改稿后**，顶栏会提示「草稿已改，可能过期」，点「重新分析」刷新

### Reviewer 2 对抗

**用途**：模拟一个苛刻的审稿人挑刺，你可以逐条反驳（rebuttal 练习）。

**操作步骤**：

1. 切到「**Reviewer 2**」子页签
2. 选择**会议**（NeurIPS / ICML / ICLR / ACL / CVPR / KDD / CHI 或通用）
3. 选择**评审风格**：
   - Reviewer 2（苛刻）— 最挑剔
   - AC（均衡）— 客观公正
   - 领域专家 — 侧重技术深度
   - 友好评审 — 建设性意见
4. 点击「**红队这篇**」
5. AI 逐条输出批评意见，每条可展开：
   - 点击「**反驳 rebuttal**」展开输入框
   - 写下你的反驳理由，点击「**发送**」
   - AI reviewer 会回复——可能被你说服，也可能继续追问
   - 来回几轮后，状态变为「已反驳」
6. 完成后点右上角「**↓ 导出 rebuttal**」下载 Markdown 文件，包含所有批评点和你的反驳

### 三角度并行评审

开启后，AI 同时从三个角度审稿并自动去重：
- **方法** — 实验设计、统计方法是否合理
- **实验** — 数据集、基线、消融实验是否充分
- **写作** — 表述清晰度、逻辑连贯性

> 在设置中启用 `features.parallel_review`，或在评审面板选择并行模式。

### 导入真实审稿意见

如果你收到了真实的 reviewer 意见，也可以导入：

1. 在 Reviewer 2 子页签底部找到「**导入真实审稿意见**」
2. 把审稿意见粘贴进文本框
3. 点击「**导入审稿意见**」
4. AI 把零散的意见结构化为可逐条反驳的条目
5. 接下来的 rebuttal 流程和模拟评审一样

## 思维导图

从编辑器欢迎页点「新建工程」→「从思维导图开始」进入，或编辑器工具栏点思维导图图标切换。

### 基本操作

| 操作 | 方法 |
|------|------|
| 添加子节点 | 选中节点 → **Tab** |
| 添加兄弟节点 | 选中节点 → **Enter** |
| 编辑节点文字 | 选中节点 → **F2** |
| 删除节点 | 选中节点 → **Delete** |
| 移动节点 | 拖拽节点卡片 |
| 删除连线 | 悬停连线变红 → 点击删除 |
| 缩放 / 平移 | 鼠标滚轮 / 拖拽画布空白处 |
| 自动布局 | 工具栏「一键整理」按钮 |

### 节点正文

每个节点除了标题，还有可折叠的**正文区域**：
- 点击节点右侧 **▸** 展开编辑区（有正文时按钮高亮）
- 收起时显示首行预览（≤40 字）
- 导出为论文大纲时，标题变成 heading，正文变成段落

### AI 辅助

- **AI 展开**：选中节点 → 点击「AI 展开」→ 自动生成子主题
- **AI 分析**：对整张图做逻辑检查，找出论证缺口

### 与编辑器同步

思维导图和编辑器可以双向切换：
- **编辑器 → 思维导图**：大纲自动解析为节点树（标题 + 正文）
- **思维导图 → 编辑器**：保留所有标题和正文内容，导出为 Markdown

## 设置与引擎配置

点击顶栏右侧状态指示器（显示 Ollama / 云端），展开服务状态面板。

### 翻译引擎

在设置面板中选择：

| 引擎 | 说明 |
|------|------|
| **Ollama（本地）** | 离线运行，无需网络，隐私最佳。需先安装 Ollama 并拉取模型 |
| **云端 API** | 支持 21 家供应商，翻译质量更高。需要填写 API Key |

支持的云端供应商：OpenAI、Anthropic、DeepSeek、Moonshot、智谱、通义千问、Gemini、SiliconFlow、OpenRouter、Groq、Together、Mistral、xAI、Fireworks、DeepInfra、Perplexity、Novita、火山方舟、百度千帆、Azure、自定义。

### 界面语言

设置面板中有语言切换下拉框，支持：
- 简体中文（zh-CN）
- English（en-US）

切换即时生效，无需重启。

### 服务状态

状态面板显示：
- 🟢 后端：API 服务是否在线
- 🟢 Ollama：本地模型是否就绪（使用 Ollama 引擎时）

如果显示离线，点击「重启后端」。

## 常见问题

### Q：启动后状态显示离线

**A**：后端服务需要几秒启动。如果等待 30 秒仍未上线：
1. 检查 18088 端口是否被占用
2. 点击「重启后端」
3. 如果使用 Tauri 桌面端，关闭后重新打开

### Q：Ollama 显示离线

**A**：
1. 确认已安装 Ollama：终端运行 `ollama list`
2. 确认模型已拉取：`ollama pull qwen3:8b`
3. Ollama 默认端口 11434，确保未被修改

### Q：不装 Ollama 能用吗

**A**：能。在设置里选「云端 API」，填上 API Key（推荐 DeepSeek，便宜好用）。本地 Ollama 只是可选项。

### Q：Windows 上启动卡住 / httpx 报错

**A**：这是 Windows 代理环境变量导致 httpx 导入卡死。使用 `start_dev.bat` 启动，它会自动清除 `HTTP_PROXY` 等环境变量。

### Q：翻译质量不好怎么办

**A**：
1. 切换到云端引擎（如 DeepSeek / GPT-4），质量显著优于本地 Ollama
2. 在设置里调低 `temperature`（如 0.2），输出更稳定
3. 利用「失败块重试」单独重翻质量差的段落

### Q：Ghost Text 不出现

**A**：Ghost Text 需要后端 `/api/complete` 接口可用。确认：
1. 后端服务在线
2. 在编辑器中停止打字后等待 1.5 秒
3. 光标在行尾（不在选区中）

### Q：Agent 不执行工具 / 总是拒绝

**A**：
1. 确认已通过「打开文件夹」指定了项目目录
2. 确认翻译引擎可用（Agent 复用翻译引擎的 LLM）
3. 尝试更明确的指令，例如 "读取 introduction.md 文件" 而非 "看看引言"

### Q：论证账本分析不出结果

**A**：
1. 确保论文中有 abstract 或 introduction 部分（AI 从这些区域提取承诺）
2. 确保翻译引擎可用（本地 Ollama 或云端 API）
3. LLM 不可用时，分析会静默失败——检查服务状态面板

### Q：导出 LaTeX 失败

**A**：
1. 检查 Tectonic 是否安装（设置中可一键安装）
2. 确认 Markdown 内容格式正确（标题层级、公式语法等）
3. 如果 Tectonic 不可用，可以导出 .tex 文件后手动用其他 LaTeX 编译器编译

### Q：切换语言后部分文字没变

**A**：切换语言即时生效，但当前打开的对话框内容不会被翻译。刷新页面即可完全切换。

### Q：语音助手无法唤醒

**A**：
1. 确认设置面板「语音」标签中已开启语音助手
2. 唤醒词仅在前台生效：把研墨窗口切换到当前活动窗口
3. 全局热键（`Alt+Shift+V`）不生效？确认桌面端版本（热键依赖 Tauri 插件，Web 端不支持）
4. 在设置里重新录制热键：点击热键输入框，按下新组合键

### Q：语音打字不工作 / 不说话

**A**：
1. 确认浏览器/WebView 已授权麦克风权限
2. 确认当前不在使用唤醒词语音助手（语音打字时唤醒词应自动暂停）
3. 检查是否有其他应用占用麦克风
4. 重启应用后重试

### Q：语音识别出现重复文字

**A**：Chrome 连续模式偶尔会重新识别之前的音频。研墨内置了三层去重机制（前缀重叠检测 + utterances 追踪 + 内部重复清洗），大部分情况下自动消除。如果仍然出现重复，点击麦克风停止录音后重新开始即可。
