## 安装与启动

### 前提条件

安装 [Ollama](https://ollama.com) 并拉取模型（本地翻译必需）：

```bash
ollama pull qwen3:8b
```

### 方式一：桌面端（推荐）

1. 从 [GitHub Releases](https://github.com/zuowen7/scholar-cursor/releases) 下载安装包
2. 安装后启动即可，应用自动管理后端服务

**Windows 开发者**：使用 `start_dev.bat` 启动（自动清除代理环境变量，避免卡顿）

### 方式二：开发模式

```bash
# 终端 1 — 启动 Python 后端
cd python
pip install -r requirements.txt
python api.py

# 终端 2 — 启动前端
npm install
npm run dev
```

浏览器打开 `http://localhost:5173`，前端自动代理 `/api` 到 `localhost:18088`。

### 方式三：Docker

```bash
docker compose up
```

## 翻译模式

顶栏切到「翻译」即可进入。这是最简单的入手点——拖入 PDF，等待完成。

### 翻译一篇论文

1. **上传文件**：点击上传区域选择 PDF，或直接把文件拖入窗口任意位置
2. **等待处理**：5 步管道自动执行，进度条实时显示
   - 解析 → 清洗 → 切块 → 翻译 → 格式化
3. **查看结果**：翻译完成后进入结果页面

### 结果视图

| 视图 | 说明 |
|------|------|
| **双语对照** | 左原文右译文，鼠标悬停某个句子，另一侧对应句子高亮 |
| **纯译文** | 只显示翻译后的文本 |
| **Markdown** | 渲染为格式化的 Markdown 页面 |

### 翻译失败重试

如果某段翻译失败（红色标记），不用重新翻译全文：
- 点击失败的段落 → 选择「重试」→ 仅重新翻译该段

### 导出翻译结果

右上角下拉菜单，4 种格式：

| 格式 | 内容 |
|------|------|
| 双语 Markdown | 中英交替，段落对齐 |
| 纯译文 Markdown | 只保留中文翻译 |
| 双语 Word (.docx) | 中英对照，适合打印 |
| 纯译文 Word (.docx) | 纯中文版本 |

## 编辑器模式

顶栏切到「编辑器」进入。核心是 Monaco 编辑器 + 右侧面板，用于写论文和 AI 辅助改稿。

### 创建文件

进入编辑器后，欢迎页提供 4 个入口：

| 入口 | 用途 |
|------|------|
| **新建工程** | 从思维导图开始，一步步生成论文草稿 |
| **从模板新建** | 选 IEEE / ACM / NeurIPS 等模板，AI 生成大纲 |
| **打开文件夹** | 打开本地已有的论文项目 |
| **新建文档** | 创建空白 Markdown 文件 |

### 文件管理

- 左侧**文件树**：浏览、新建、删除文件
- 顶部**标签栏**：多文件切换，拖拽排序，`Ctrl+W` 关闭当前标签

### AI 对话面板

右侧面板切到「AI 编辑」，这是一个聊天式 AI 助手：

**快速操作**（底部预设按钮）：

| 按钮 | 效果 |
|------|------|
| 润色 | 改善选中段落的行文质量 |
| 扩写 | 将选中段落展开为更详细的论述 |
| 审查 | 检查逻辑漏洞、论证不足之处 |
| 英译 | 将选中中文翻译为学术英文 |
| 中译 | 将选中英文翻译为中文 |

**使用方式**：
1. 在编辑器里**选中一段文字**
2. 点击预设按钮（或在输入框直接打字提要求）
3. AI 返回结果，点击「**插入**」替换选中文字，或「**复制**」到剪贴板
4. 不满意？点击「**撤销**」恢复原文

**高级用法**：
- 输入 `/` 呼出斜杠命令菜单
- 输入 `@` 引用工作区其他文件作为上下文
- 点击 📎 按钮添加附件

### Ghost Text（自动续写）

在编辑器里打字后**停顿 1.5 秒**，AI 会在光标处生成灰色续写建议：
- **Tab** — 接受建议
- **Esc** — 忽略

### 预览

右侧面板切到「预览」，实时渲染当前 Markdown 为排版效果，所见即所得。

## 论证陪练

右侧面板切到「论证陪练」，这是投稿前的自查工具，分两个子页签。

### 论证账本

**用途**：AI 自动检查你在 abstract/intro 里许下的每一条承诺，是否在正文中实际兑付了。

**操作步骤**：

1. 打开你的论文文件
2. 点击「**分析论证账本**」按钮
3. AI 扫描后，每条承诺显示为一行：

| 状态 | 含义 |
|------|------|
| ✅ 已兑付 | 正文有对应实验/论证支撑 |
| ⚠️ 部分兑付 | 有支撑但不充分 |
| ❌ 未兑付 | 正文找不到对应的实验/论证 |
| ⚠️ 不一致 | 兑现了但和承诺的内容对不上 |

4. **点击承诺文本** → 编辑器跳转到对应原文位置
5. 点击「**→ 兑付处**」→ 跳到正文中支撑该承诺的位置
6. 对未兑付的承诺点「**怎么补满**」→ AI 给出具体实验设计建议
7. **改稿后**，顶栏会提示「草稿已改，可能过期」，点「重新分析」刷新

### Reviewer 2 对抗

**用途**：模拟一个苛刻的审稿人挑刺，你可以逐条反驳（rebuttal 练习）。

**操作步骤**：

1. 切到「**Reviewer 2**」子页签
2. 选择**会议**（NeurIPS / ICML / ICLR / ACL / CVPR / KDD / CHI 或通用）
3. 选择**评审风格**：
   - Reviewer 2（苛刻）— 最挑剔
   - AC（均衡）— 客观公正
   - 领域专家 — 侧重技术深度
   - 友好评审 — 建设性意见
4. 点击「**红队这篇**」
5. AI 逐条输出批评意见，每条可展开：
   - 点击「**反驳 rebuttal**」展开输入框
   - 写下你的反驳理由，点击「**发送**」
   - AI reviewer 会回复——可能被你说服，也可能继续追问
   - 来回几轮后，状态变为「已反驳」
6. 完成后点右上角「**↓ 导出 rebuttal**」下载 Markdown 文件，包含所有批评点和你的反驳

### 导入真实审稿意见

如果你收到了真实的 reviewer 意见，也可以导入：

1. 在 Reviewer 2 子页签底部找到「**导入真实审稿意见**」
2. 把审稿意见粘贴进文本框
3. 点击「**导入审稿意见**」
4. AI 把零散的意见结构化为可逐条反驳的条目
5. 接下来的 rebuttal 流程和模拟评审一样

## 思维导图

从编辑器欢迎页点「新建工程」→「从思维导图开始」进入，或编辑器工具栏点思维导图图标切换。

### 基本操作

| 操作 | 方法 |
|------|------|
| 添加子节点 | 选中节点 → **Tab** |
| 添加兄弟节点 | 选中节点 → **Enter** |
| 编辑节点文字 | 选中节点 → **F2** |
| 删除节点 | 选中节点 → **Delete** |
| 移动节点 | 拖拽节点卡片 |
| 删除连线 | 悬停连线变红 → 点击删除 |
| 缩放 / 平移 | 鼠标滚轮 / 拖拽画布空白处 |
| 自动布局 | 工具栏「一键整理」按钮 |

### AI 辅助

- **AI 展开**：选中节点 → 点击「AI 展开」→ 自动生成子主题
- **AI 分析**：对整张图做逻辑检查，找出论证缺口

### 导出

思维导图可以一键转为论文大纲，再进入编辑器继续写作。

## 设置与引擎配置

点击顶栏右侧状态指示器（显示 Ollama / 云端），展开服务状态面板。

### 翻译引擎

在设置面板中选择：

| 引擎 | 说明 |
|------|------|
| **Ollama（本地）** | 离线运行，无需网络，隐私最佳。需先安装 Ollama 并拉取模型 |
| **云端 API** | 支持 21 家供应商，翻译质量更高。需要填写 API Key |

支持的云端供应商：OpenAI、Anthropic、DeepSeek、Moonshot、智谱、通义千问、Gemini、SiliconFlow、OpenRouter、Groq、Together、Mistral、xAI、Fireworks、DeepInfra、Perplexity、Novita、火山方舟、百度千帆、Azure、自定义。

### 服务状态

状态面板显示：
- 🟢 后端：API 服务是否在线
- 🟢 Ollama：本地模型是否就绪（使用 Ollama 引擎时）

如果显示离线，点击「重启后端」。

## 常见问题

### Q：启动后状态显示离线

**A**：后端服务需要几秒启动。如果等待 30 秒仍未上线：
1. 检查 18088 端口是否被占用
2. 点击「重启后端」
3. 如果使用 Tauri 桌面端，关闭后重新打开

### Q：Ollama 显示离线

**A**：
1. 确认已安装 Ollama：终端运行 `ollama list`
2. 确认模型已拉取：`ollama pull qwen3:8b`
3. Ollama 默认端口 11434，确保未被修改

### Q：Windows 上启动卡住 / httpx 报错

**A**：这是 Windows 代理环境变量导致 httpx 导入卡死。使用 `start_dev.bat` 启动，它会自动清除 `HTTP_PROXY` 等环境变量。

### Q：翻译质量不好怎么办

**A**：
1. 切换到云端引擎（如 DeepSeek / GPT-4），质量显著优于本地 Ollama
2. 在设置里调低 `temperature`（如 0.2），输出更稳定
3. 利用「失败块重试」单独重翻质量差的段落

### Q：Ghost Text 不出现

**A**：Ghost Text 需要后端 `/api/complete` 接口可用。确认：
1. 后端服务在线
2. 在编辑器中停止打字后等待 1.5 秒
3. 光标在行尾（不在选区中）

### Q：论证账本分析不出结果

**A**：
1. 确保论文中有 abstract 或 introduction 部分（AI 从这些区域提取承诺）
2. 确保翻译引擎可用（本地 Ollama 或云端 API）
3. LLM 不可用时，分析会静默失败——检查服务状态面板

### Q：导出 LaTeX 失败

**A**：
1. 检查 Tectonic 是否安装（设置中可一键安装）
2. 确认 Markdown 内容格式正确（标题层级、公式语法等）
3. 如果 Tectonic 不可用，可以导出 .tex 文件后手动用其他 LaTeX 编译器编译
